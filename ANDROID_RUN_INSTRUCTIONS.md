# 🚀 ЗАПУСК ANDROID ПРИЛОЖЕНИЯ - ПОЛНАЯ ИНСТРУКЦИЯ

## ✅ ЧТО НУЖНО СДЕЛАТЬ ДО ЗАПУСКА:

### 1️⃣ ЗАПУСТИТЬ BACKEND (КРИТИЧНО!)

Backend должен быть запущен, иначе приложение не будет работать!

```bash
cd backend-spring
mvn spring-boot:run
```

Ждите пока напишет:
```
Started CodeCraftBackendApplication in X.XXX seconds
```

### 2️⃣ ПРОВЕРИТЬ ЧТО BACKEND РАБОТАЕТ

В **другом терминале**:
```bash
curl http://localhost:8080/api/health
```

Должны получить:
```json
{"status":"ok","supabase":"connected"}
```

✅ Если получили - backend работает!

## 🔧 КОНФИГУРАЦИЯ ANDROID ПРИЛОЖЕНИЯ

### Если тестируете на ЭМУЛЯТОРЕ (локально):

Приложение уже настроено использовать `http://10.0.2.2:8080/api`
(это специальный адрес для доступа к localhost от эмулятора)

**ВСЕ РАБОТАЕТ БЕЗ ИЗМЕНЕНИЙ!** ✓

### Если тестируете на РЕАЛЬНОМ УСТРОЙСТВЕ:

Нужно изменить URL backend'а в коде:

1. Откройте `app/src/main/java/com/prolearn/codecraftfront/data/LeaderboardRepository.kt`

2. Найдите строку:
   ```kotlin
   private val backendUrl = "http://10.0.2.2:8080/api"
   ```

3. Замените на ваш реальный адрес:
   ```kotlin
   private val backendUrl = "https://your-backend-domain.com/api"
   ```

4. Повторите в `app/src/main/java/com/prolearn/codecraftfront/service/LevelService.kt`

## 📱 ЗАПУСК НА ЭМУЛЯТОРЕ (РЕКОМЕНДУЕТСЯ)

### Шаг 1: Откройте Android Studio

`Файл` → `Открыть` → выберите `CodeCraftFrontApp`

### Шаг 2: Дождитесь синхронизации Gradle

Android Studio может попросить обновить плагины - нажмите "Update"

### Шаг 3: Запустите эмулятор

```

Нажмите "Launch"
```

Ждите пока эмулятор запустится (~30 сек)

```
Нажмите: Run (зеленая кнопка играющего)
Или: Shift + F10
```

Приложение установится и запустится на эмуляторе.

## 🔍 ЧТО ПРОВЕРИТЬ:

### ✅ Приложение работает если:
```
1. Запустилось без ошибок
2. Firebase вход работает
3. Видите свой профиль (XP, Level)
4. Видите список уровней
5. Можете написать код
6. После отправки видите результат (правильно/неправильно)
7. XP обновился в профиле
```

### ❌ Проблемы могут быть если:

#### "Connection refused" в логах
→ Backend не запущен
→ Запустите: `cd backend-spring && mvn spring-boot:run`

#### "Invalid token" ошибка
→ Firebase токен неправильный или истек
→ Перезагрузитесь в приложении, заново авторизуйтесь

#### "Supabase connection refused"
→ Интернет не включен на эмуляторе
→ Проверьте что Wi-Fi разрешен эмулятору (Settings → Wireless)

#### XP не обновляется после решения
→ Backend не получил запрос
→ Проверьте логи в терминале backend'а
→ Убедитесь что используется правильный URL
